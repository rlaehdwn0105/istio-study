## 공식 문서

### 강점

- 현재 기준에서 가장 정확하다
- 설치 절차와 개념 정의의 기준점 역할을 한다
- 이후 주차까지 연결되는 용어 체계가 안정적이다

### 약점

- 초심자에게는 맥락 설명이 덜 친절할 수 있다
- 왜 이런 기능이 필요한지 운영 서사를 충분히 주지 않는 경우가 있다

### 용도

- 명령어와 최신 동작 검증
- 최신 용어 정리
- 버전 차이 보정

## kschoi728

### 강점

- 1주차가 매우 촘촘하게 분해되어 있다
- 개념, 설치, control plane, 샘플 앱, observability, resiliency, routing까지 이어지는 흐름이 좋다
- 한국어로 읽기 편하고 주차별 소화가 쉬운 편이다

### 약점

- 버전 기준이 과거 스터디에 맞춰져 있다
- 일부 실습은 현재 릴리스와 차이가 있을 수 있다

### 용도

- 1주차 메인 참고 블로그
- 학습 순서 설계
- 개념 설명 보강

## kimdoky

### 강점

- 도식과 이미지가 풍부하다
- 학습 동선을 한 페이지에서 잡기 좋다
- 복습 자료나 발표 자료처럼 쓰기 좋다

### 약점

- 한 글에 내용이 많아 처음 보면 밀도가 높게 느껴질 수 있다

### 용도

- 구조 시각화
- 1주차 복습용
- 이미지 자산 수집

## zerotay

### 현재 상태

- Notion에서는 추천되지만, 현재 공개 링크는 직접 접근 시 `Nothing here` 상태다

### 처리

- 출처 목록에는 남겨둔다
- 실제 학습용 핵심 본문은 접근 가능한 자료 위주로 구성한다

## 1주차에서의 권장 사용법

1. 공식 문서로 현재 기준 개념과 설치 흐름 확인
2. `kschoi728`로 한국어 개념과 주차 세분화 이해
3. `kimdoky`로 그림 중심 복습
4. 실습은 `practice/` 폴더의 YAML로 재현
